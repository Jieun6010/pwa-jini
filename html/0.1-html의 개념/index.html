Html Hyper Markeup Language

웹문서의 DOM구조를 설계하는 마크업 언어를 의미한다.

html문서의 특징

  • 요소(element), 태그(tag), 속성(attribute)이 존재한다.
  • Node(parent(부모), children(자식), decendent(자손), siblings(동급/형제자매))가 존재한다.
  • Node는 정해진 구조를 준수해야 한다.

시멘틱 요소

1. 구획(범위)요소 / 영역을 나누는 !

  • main : 웹문서의 주요내용
  • section, article : 제목과 내용을 포함하는 주제를 출력하는 영역
  • header : 주로 웹문서 상단의 로고나 정체성을 나타내고 표현하는 영역
  • footer : 주로 웹문서 하단부분에 표현 / 사이트의 보조적인 정보를 출력하는 영역
  • aside : 사이트의 부수적인 정보나 기능을 출력하는 영역 / 사이드 바 (광고도 포함)
  • nav : 메뉴 기능을 제공하는 영역
  • div : 레이아웃 영역(무언가 들어가있지는 않지만 아무거나 넣을 수 있음)

2. 다양한 시멘틱 요소(의미가 있다는 ! 구조적인)

  • em : 중요함을 나타내고 싶을 때
  • strong, mark : 시각적인 강조를 의미
  • small : 보조적인 의미
  • i : 구분, 다름(차이)의 목적
  • blockquote :
    다른 사람의 글을 퍼올 때 / 인용
  • cite : 인용처(출처 : mbc)
  • span : 의미가 없음을 의미 / 주로 디자인때문에 사용
  • time : 시간표시
  • address : 주소, 전화, (주로 웹사이트에서 회사 번호나 주소 위치 등을 표시할 때)
  • abbr : 약어 / 예 : 제 이름은 박지은인데 pje로 불러주세요
  • dfn : 용어 /예 : pje는 박지은을 의미해요

3. 물리적 요소 ( 시각적인 변화 ) / css가 있기때문에 굳이 쓰지 않아도 되는 !

  • b : 두꺼운 글자
  • u : 밑줄
  • strike : ㅇ...
  • sup : 윗첨자 이건 써야함 22 (2의 2승..)
  • sub : 아래첨자 이것두 쓰긴 하겠지 !
