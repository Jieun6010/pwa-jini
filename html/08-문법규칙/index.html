syntax rule

kldp html 유효성(validator) 검사를 이용해 확인해 볼 것

문법규칙(규칙은 무조건 법!)

  • inline요소의 자식으로는 block요소는 허락되지 않는다.
    예외 : a는 interactive를 제외한 모든 요소의 부모가 될 수 있다.
  • heading요소, p요소는 인라인레벨만 허락된다.
  • interactive요소(폼요소,버튼,셀렉트,인풋,앵커 등)간에는 부모자식관계가 허락되지 않는다
  • figcaption의 자식으로는 a요소가 허락되지않는다.
  • ul, ol의 자식으로는 li만 허락된다.
  • li의 자식으로는 모든요소가 허락된다.
  • dl의 자식으로는 dt, dd요소만 허락된다.
  • style, meta, title요소는 head의 자식요소로만 허락된다.
  • img요소는 alt속성을 가져야만 한다.

권장사항(지켜도 되고 안지켜도 되는,,, 노란색 경고가 뜬다.)

  • heading요소는 텍스트를 포함하길 권장
  • h1과 main요소는 웹문서에 한개씩 포함하길 권장
  • main, section, article요소는 heading요소를 한개 이상 포함하길 권장
  • heading요소는 순차적으로 적용하길 권장(h1에서 갑자기 h3로 튀어나가지않게..되도록이면!, 실질적으로 지키기는 힘들다.)
  • title값은 page마다 다르게 출력하길 권장한다.
  • WAI-ARIA 규칙을 준수하길 권장
